# BDC Web App

A tournament management platform for Bannerlord Draft Cups. Handles the full lifecycle: player auctions, team drafting, match scheduling, pick-bans, and public-facing stats.

For a detailed breakdown of the data model, API routes, and component architecture, see [spec.md](spec.md).

---

## Roles

### Admin

Logs in at `/admin-login`. Responsible for platform-wide configuration.

- **Hosts**: Create and manage host accounts (username/password). Hosts use these credentials to log in and manage their tournaments.
- **Maps**: Maintain the global map pool used across all tournaments.
- **Featured tournament**: Mark one tournament as "featured" — it appears prominently on the home page.
- **Division stats**: Upload `.csv` or `.json` stat files per division. After uploading, configure column visibility, conditional highlights, color gradients, and player name mappings via the settings (gear) icon. Mark captains with the crown toggle. Import settings from another division to reuse the same column config quickly.
- **Player overrides**: Adjust individual player prices or other auction values after the fact.

---

### Host

Logs in at `/login`. Manages one or more tournaments end-to-end.

**Typical workflow:**

1. **Create a tournament** — set a name and description.
2. **Create divisions** — each division runs its own auction (e.g. "Division 1", "Division 2").
3. **Add players to each division's auction** — paste in player names and set their starting prices.
4. **Assign an auctioneer** — generate a token for the auctioneer to use during the live session.
5. **Run the auction** — the auctioneer runs the live bidding; captains join with their own tokens.
6. **Review teams** — after the auction, view resulting rosters per division.
7. **Set up the schedule** — create matchups for group stage rounds.
8. **Configure pick-bans** — for each matchup (and especially finals), set up the pick-ban map pool.
9. **Mark results** — record match scores and winners.
10. **Manage the bracket** — advance teams through playoff rounds.

Hosts can also set team names and edit rosters directly from the tournament admin panel.

---

### Auctioneer

Accesses the live auction at `/auction/:token` using a token generated by the host.

- Sees the full player pool and current bids.
- Controls auction pacing: start bidding on a player, accept a bid, move to the next player.
- The live session is visible to captains in real time.

---

### Captain

Accesses their team view at `/captain/:token` using a token generated by the host.

- Participates in live bidding during the auction phase.
- After the auction: views their roster, sets or updates the team name.
- During pick-ban phase: interacts with the pick-ban interface to select maps or sides (depending on the configured sequence).

## Public Pages

No login required.

| Route | Description |
|---|---|
| `/` | Home page — shows the featured tournament and general info |
| `/tournaments` | List of all tournaments |
| `/tournaments/:slug` | Tournament detail page — divisions, standings, schedule, stats |
